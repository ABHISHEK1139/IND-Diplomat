Step 2 — Filter noise: don't trust a single source

belief_accumulator.py

Raw data isn't treated as truth. Before any analysis happens, the system requires independent confirmation — one blog post mentioning "troops" doesn't move the needle.

Raw Data
A news article or dataset entry
→
Observation
"This might be happening"
→
Confirmed Signal
Multiple independent sources agree
🔁Duplicate detection — If 5 outlets republish the same Reuters report, it counts as 1 source, not 5
⏱Freshness check — A 3-day-old report has half the weight of a fresh one. After a week, almost none.
📡Multi-source bonus — When GDELT and SIPRI both detect the same signal independently, confidence increases
🏷Source trust tiers — Satellite data (0.90) is weighted more than social media (0.30)
This prevents the Iraq WMD problem: many reports citing the same original source being treated as independent confirmation.

Step 3 — Classify: what state is this country in?

conflict_state_model.py

The model places each country on a 5-state scale from Peace to Full War. It doesn't pick a single label — it gives a probability distribution across all states, so the analyst can see exactly how uncertain the system is.

Remembers history — Each run's output becomes the next run's starting point, so the model tracks how a situation evolves over time
No sudden jumps — A country can't go from Peace to War in one step. The transition matrix enforces realistic state changes
Self-correcting — When outcomes resolve, the model updates its own transition probabilities to get better over time

Step 5 — Challenge: is the analysis actually correct?

red_team.py + cove.py + crag.py
⚠ NOT ROBUST
−6.0% confidence penalty applied

Before any assessment is released, three verification systems stress-test it:

Red Team — Challenges the assessment across 6 dimensions. Found only 2 sources here — applied a confidence penalty.
Claim Verification — Breaks the assessment into individual claims and checks each one against the evidence.
Retrieval Quality Check — Evaluates whether the evidence retrieved was actually relevant to the question asked.
Finding: Diplomacy (0.95) and Hostility (0.95) are both high — the system flags this as possible coercive diplomacy, not a contradiction.

Step 6 — Gate: should this assessment be released at all?

assessment_gate.py

This is the most important part. The gate runs 5 hard-coded rules — no AI, no randomness. If the evidence is too thin, the system says "I don't know" instead of guessing. It must earn the right to release an assessment.

1Enough data?System hasn't requested too many follow-up collection tasks✓ PASS
2Military coverage?Must have at least 35% military data to assess escalation✓ PASS
3Data is fresh?Rejects assessments based on outdated intelligence✓ PASS
4Confident enough?Overall confidence must be above 55%✓ PASS (63%)
5Trend checkIf signals are rising fast but assessment says "low" — override it↑ +MEDIUM
APPROVED — Assessment released
In high-stakes systems, you can't let an AI model decide what gets published. These rules are deterministic — the same input always gives the same decision. No hallucination possible.
