Step 1 — Data collected from 0 sources

Layer1_Collection/

The system pulled data from independent geopolitical databases — arms trade records, conflict events, economic indicators, treaty registries, and live news.

Each provider maps raw data into signal groups (military, diplomatic, economic, etc.). Siloed datasets are fused into a single analytical ontology.

Step 2 — Confirmed signals after noise filtering

belief_accumulator.py

Raw data isn't treated as truth. The system requires independent confirmation before a signal moves the analysis. Duplicate detection, freshness checks, and multi-source triangulation are applied.

SignalConfidence
This prevents single-source bias — many reports citing the same original source are collapsed into one observation. Only independently confirmed signals pass through.

Step 3 — Conflict state classification

conflict_state_model.py

The model places the country on a 5-state scale. It doesn't pick a single label — it gives a probability distribution so the analyst sees exactly how uncertain the system is.

Strategic Risk Decomposition (SRE)

Step 4 — AI analysts debated the evidence

coordinator.py

Each "minister" looks at the same data through a different lens. They independently propose risk assessments, then the system synthesizes their views — weighting by confidence and reasoning strength.

Multiple perspectives reduce blind spots. Disagreements are flagged and investigated further.

Step 5 — Verification: is the analysis correct?

red_team.py + cove.py + crag.py
Before any assessment is released, three verification systems stress-test it. Red Team challenges across 6 dimensions, CoVe verifies individual claims, CRAG checks retrieval quality.

Step 6 — Gate: should this assessment be released?

assessment_gate.py

The gate runs hard-coded rules — no AI, no randomness. If evidence is too thin, the system says "I don't know" instead of guessing. It must earn the right to release.

In high-stakes systems, AI can't decide what gets published. These rules are deterministic — same input always gives the same decision. No hallucination possible.
